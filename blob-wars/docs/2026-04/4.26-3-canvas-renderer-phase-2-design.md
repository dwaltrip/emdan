# Canvas renderer: Phase 2 design

Concrete design for Phase 2: the tile layer (offscreen canvas), continuous-rAF, dirty-flag split, and seed-pulse. Supersedes `4.26-2-canvas-renderer-phase-2-prep.md` — the prep doc captured framings; this doc captures decisions we can implement from.

**Terminology shift.** The prep doc and Phase 1 sketch refer to the offscreen tile cache as a "buffer." This doc adopts **"layer"** instead — accurate to the layered-composition pattern (no swap semantics, no double-buffering). Older docs still use the outdated term; treat this one as canonical.

## Key concepts / terminology

### Render artifacts

- **Layer.** An offscreen bitmap holding one or more passes' output, composited onto the main canvas each frame. Today: one (`tileLayer`). Distinct from "buffer" (no swap).
- **Frame.** One rendered output of the rAF loop on the main canvas. Data noun behind `frameDirty` and `renderFrame`.
- **Composite.** Per-frame action of blending layers + effects onto the main canvas to produce a frame.

### Function-of-input categories

Four cells from the cross-product of "depends on input?" × "depends on time?":

| Category | Signature | Examples |
|---|---|---|
| Tile pass | `(state)` | tiles, walls, exclusion fill |
| Pure effect | `(state, input)` | hover ring, exclusion preview |
| Time effect | `(state, t)` | seed-pulse |
| Input + time effect | `(state, input, t)` | future: animated hover-conditional indicators |

### Frame-input slots

Three slots, distinct lifetimes, uniform per-frame read at the top of `renderFrame`:

- **Game state.** Long-lived; snapshot-replaced on tick. Lives in store.
- **Input state.** Ephemeral; mutated by event handlers. Renderer-local closure (`hoverState` today; modifier keys, drag origin, dwell time tomorrow). Presentation concern, not a game concern.
- **Time progress.** Continuous; mutates every frame alive. Renderer-local closure (`pulseStart` later).

### Control state

- **`tilesDirty`.** Tile-pass output is stale; redraw to `tileLayer` next frame.
- **`frameDirty`.** Frame on main canvas is stale; recomposite next frame.
- **`requestTilesRedraw`.** Caller API; sets both flags.
- **`requestComposite`.** Caller API; sets only `frameDirty`.
- **`animating`.** Derived predicate, `true` while any time effect is in flight; gates continuous-rAF.

### Patterns

- **Time-effect interface.** `trigger / isActive / draw`. Module owns closure-scoped state; renderer drives it from `subscribe` and `renderFrame`.
- **Slice-change-triggered effect.** A class of effect fired by diffing a slice of game state across consecutive store notifies (e.g., `tick` advance → pulse).

## Key architectural decisions & insights

- **Layering, not perf, is the layer's purpose.** Phase 1 measured <5ms full redraw — `tileLayer` adds *structure* (static layer under dynamic layers); perf is a side-effect win. Drives downstream calls (the dirty-flag split into two flags is justified by layering, not perf).
- **Time is orthogonal to input.** The four function-of-input cells are the cross-product of two binary axes. Today we use 3 of 4; the 4th `(state, input, t)` is comment-only.
- **Tile pass earned its own row by size and cacheability, not function-of-input distinctness.** It's *also* a pure projection (pure-of-time, pure-of-input). What separates it is that it's big and stable enough to cache as a layer.
- **Single-timestamp pattern with a documented boundary.** Seed-pulse uses one global `pulseStart`. Per-instance timestamps (entity territory) only become necessary at (mid-sim instance changes) ∧ (overlap-aware visuals). We have neither today; comment in `seed-pulse.ts` records this.
- **Slice-of-state-change is a named category, not a runtime abstraction.** Inline diffing today (N=1: tick advance). Examples that fit the pattern: capture flare on owner flip, phase-change transitions, score blips. Comment names the category; abstract only when N or predicate complexity demands.
- **Time-effect interface is a candidate, not enforced.** `seed-pulse.ts` is the first module implementing it. Validate the shape at N=2 (next time effect — capture flare etc.); copy-paste before abstracting. No animation manager, no registry.
- **Continuous-rAF lives off the `scheduleFrame` / `requestComposite` split.** `if (animating) scheduleFrame()` at end of `renderFrame` self-perpetuates without a fake dirty bit. The Phase 1 split was the load-bearing prep.
- **Entity pass remains a comment-only future fourth category.** Surface in `seed-pulse.ts`'s top comment: "see also: entity pass when per-instance TTL state becomes unavoidable (capture flares, projectiles)."
- **Closure-scoped state for all per-init resources.** Renderer state, time-effect state. Per-init lifecycle, GC'd with `destroy`, no module-level mutables. `createPulseEffect()` factory mirrors `init()`'s closure pattern at smaller scope.

## Layer + frame loop architecture

### Module structure addition

```
ui/src/board-canvas/
  effects/
    seed-pulse.ts          # new: time effect, heartbeat on tick
  ...
```

`effects/` is broad enough to absorb future pure effects (e.g., exclusion preview) too — not committing to extracting `drawHoverRing` from `draw.ts` today (that's still the N=2 question from the prep doc, deferred until exclusion preview's input shape is known).

### `tileLayer` allocation

```ts
let layout = computeLayout(container, width, height);
const tileLayer = new OffscreenCanvas(layout.backingWidth, layout.backingHeight);
const tileLayerCtx = tileLayer.getContext('2d');
if (!tileLayerCtx) throw new Error('[board-canvas] tile layer 2D context unavailable');
```

Allocated in `init()`'s closure. Sizing on resize/DPR change happens via `applyLayoutToSurfaces` (next subsection).

### Layout application

`layout.ts` is purified: `applyLayout` is renamed **`computeLayout`** (pure — returns `LayoutState`; no DOM mutation). Observers (`observeContainer`, `observeDpr`) stay unchanged.

The renderer owns applying layout to its surfaces via a closure-scoped helper:

```ts
// TODO: revisit this abstraction if more layers arrive or layout complexity
// grows further — likely extract per-surface apply helpers into layout.ts at
// that point.
function applyLayoutToSurfaces(): void {
  canvas.width = layout.backingWidth;
  canvas.height = layout.backingHeight;
  canvas.style.width = `${layout.backingWidth / layout.dpr}px`;
  canvas.style.height = `${layout.backingHeight / layout.dpr}px`;
  tileLayer.width = layout.backingWidth;
  tileLayer.height = layout.backingHeight;
}

function relayout(): void {
  layout = computeLayout(container, width, height);
  applyLayoutToSurfaces();
  requestTilesRedraw();
}
```

Both surfaces are resized synchronously in the same call frame — no rAF or other render path can interleave between them. The one-frame flash window from Phase 1 still applies (between resize and the next composite); accepted as before. Future offscreen layers are absorbed by adding two lines to `applyLayoutToSurfaces` — one place, hard to forget.

### What lives where

- **`tileLayer`** (rebuilt on `tilesDirty`): tile-pass output — gutter fill, empty tiles, walls, owned base, seed white-dot, exclusion overlay.
- **Main canvas** (recomposited on `frameDirty` or `animating`): blit `tileLayer` → `drawHoverRing` (pure effect) → `pulse.draw` (time effect).

No `clearRect` on the main canvas — `drawImage(tileLayer, 0, 0)` covers the full backing area.

### Dirty flags + invariant

```ts
let tilesDirty = true;
let frameDirty = true;
```

**Invariant**: `tilesDirty → frameDirty` (rebuilding the tile layer always requires recompositing).

### Caller API split

```ts
function requestTilesRedraw(): void { tilesDirty = true; frameDirty = true; scheduleFrame(); }
function requestComposite(): void   { frameDirty = true; scheduleFrame(); }
```

Wiring:

| Trigger | Calls |
|---|---|
| `store.subscribe` (any notify) | `requestTilesRedraw` |
| `pointermove` (coord changed) / `pointerleave` | `requestComposite` |
| Layout change | `relayout` → `computeLayout` + `applyLayoutToSurfaces` + `requestTilesRedraw` |
| `pulse.trigger` (in subscribe callback) | `requestComposite` (then continuous-rAF self-perpetuates) |

**Rebuild on every store notify**, no slice-filter. Cheap (<5ms), notifies are rare (~1Hz), and in practice every notify *does* affect tiles (tick = tile flips, placement = tile flip, phase change wants a redraw anyway). Slice-watch only if evidence later calls for it.

### `renderFrame`

```ts
function renderFrame(): void {
  rafId = null;
  const now = performance.now();
  const animating = pulse.isActive(now);
  if (!tilesDirty && !frameDirty && !animating) return;

  if (tilesDirty) {
    const tick = store.state.game.tick;
    perfLog.timed('canvasDraw', tick, () =>
      drawAllTiles(tileLayerCtx, store, theme, layout),
    );
    tilesDirty = false;
  }

  // Composite: blit layer, then pure effects, then time effects.
  ctx.drawImage(tileLayer, 0, 0);
  const interaction = describeInteraction(hoverState.coord, store, session);
  canvas.style.cursor = interaction.kind === 'placeable' ? 'pointer' : 'default';
  drawHoverRing(ctx, hoverState.coord, interaction, theme, layout);
  pulse.draw(ctx, store, theme, layout, now);

  perfLog.rAFEvent('paint', store.state.game.tick);
  frameDirty = false;
  if (animating) scheduleFrame();
}
```

The rAF callback is renamed from `frame` to `renderFrame` for verb-prefix consistency with `drawAllTiles`, `computeLayout`, etc., and to disambiguate from `frameDirty`.

### `Ctx2D` type alias

```ts
// in draw.ts
type Ctx2D = CanvasRenderingContext2D | OffscreenCanvasRenderingContext2D;
```

Used by `drawAllTiles` and `drawTile` (the latter is called from `drawAllTiles` with the same ctx, so its parameter type widens too). `drawHoverRing` and `drawPulses` only ever receive the main `ctx`, so they stay typed `CanvasRenderingContext2D`.

### perfLog

`perfLog.timed('canvasDraw', ...)` scope shifts to wrap the `tileLayer` rebuild. Same semantics as Phase 1 (dominant per-tick cost). `perfLog.rAFEvent('paint', tick)` continues to fire at the end of every composite.

## Seed-pulse design

### Trigger

```ts
let prevTick = store.state.game.tick;
const unsubStore = store.subscribe(() => {
  const { phase, tick } = store.state.game;
  if (phase === 'simulating' && tick !== prevTick) {
    pulse.trigger(performance.now());
  }
  prevTick = tick;
  requestTilesRedraw();
});
```

Initialized with the snapshot's tick → no spurious initial pulse. Pulse fires only on tick advance during `'simulating'` phase.

### Single global timestamp

`pulseStart: number | null` in the closure. New tick truncates the in-flight pulse uniformly across all seeds. Coherent because seeds are frozen during sim phase. Per-seed timestamps would be entity territory; not needed today.

A pulse triggered near a phase exit can keep firing for up to `PULSE_DURATION_MS` after the phase changes — `isActive(now)` doesn't gate on phase. Intended; visually benign because `pulse.draw` iterates current `derived.seeds`, which may simply differ in `'placing'` / `'ended'`.

### Parameters

| Parameter | Value |
|---|---|
| `PULSE_DURATION_MS` | `400` (constraint: must stay below tick interval, currently 1000ms) |
| `PULSE_MAX_RADIUS_MULT` | `2.0` (radius reaches ~2 tiles at end of pulse) |

### Geometry

- **Filled circle** (no stroke), origin at `tileCenter(coord)`.
- Radius starts at `cellDevicePx * 0.15` (matches existing seed-dot), expands to `cellDevicePx * 2.0`.
- **No clipping** — pulse extends over neighbors deliberately.

### Color

- Pale, partially transparent tint of player color.
- New per-seat field `pulseFill` in `SEAT_COLORS` (e.g., `'rgba(184, 156, 255, 0.30)'` for player1).
- Per-frame fade applied via `ctx.globalAlpha = (1-p)²`, scaling the peak `pulseFill` opacity.

### Curve

```ts
const eased = 1 - (1 - p) ** 2;   // radius: fast start, slow end
const alpha = (1 - p) ** 2;       // opacity: faster decay than radius
```

Alpha decaying faster than radius gives the "puff of smoke" feel.

### Module shape

`effects/seed-pulse.ts` exports `createPulseEffect()` returning a `TimeEffect`-shaped object:

```ts
interface TimeEffect {
  trigger(now: number): void;
  isActive(now: number): boolean;
  draw(
    ctx: CanvasRenderingContext2D,
    store: BoardStoreInstance,
    theme: Theme,
    layout: LayoutState,
    now: number,
  ): void;
}
```

Closure-scoped `pulseStart`. No dispose needed (single primitive in closure; nothing to clean).

### Seed iteration

`pulse.draw` iterates `store.derived.seeds` (see Store-side change below), reading `owner` per seed for `theme.SEAT_COLORS[owner].pulseFill`.

## Store-side change

Add `seeds` to `DerivedState`:

```ts
interface DerivedState {
  excludedCoords: Map<CoordKey, Set<PlayerId>>;
  seeds: Array<{ coord: Coord; owner: PlayerId }>;
}
```

Populated in `derived.ts` via a small `collectSeeds(state)` helper. Refactor `computeExcludedCoords` to take the seed list as input (avoids walking the grid twice).

Flat array (not grouped by owner) — pulse iterates all seeds and dispatches per-seat color via `seed.owner`. Grouping isn't a current need.

Recomputed on every store notify alongside `excludedCoords`. Same cost profile as today's derive; not new pressure.

Lives in **commit 2** (alongside the renderer consumer); store change without a consumer would be churn.

## Theme additions

Add `pulseFill` to `SeatPalette`:

```ts
interface SeatPalette {
  base: string;
  hoverBorder: string;
  hoverGlow: string;
  pulseFill: string;  // new: pale, semi-transparent tint of player color
}
```

Hand-tuned per seat, no helper. Add a `TODO:` comment above `SEAT_COLORS` flagging that as the file grows or aesthetics iterate, switch to deriving variants from `base` via a color helper (candidate: `colord` — ~3KB, no deps, modern API).

## Code comments to encode

Specific comments to write into the new code (separate from this design doc):

- **`effects/seed-pulse.ts` top comment**:
  - Module is the first instance of the time-effect interface — copy this shape for the next time effect, validate at N=2, abstract only if needed.
  - Single-timestamp boundary: per-seed timestamps would only matter at (mid-sim instance changes) ∧ (overlap-aware visuals). Neither holds.
  - See also: entity pass becomes the natural home when per-instance TTL state is unavoidable (capture flares, projectiles).
- **`effects/seed-pulse.ts` near constants**: list of alt curves to explore (cubic-bezier expansion, additive blend on rapid ticks, ring-stroke variant, per-seat asymmetric curves). One line, not a manifesto.
- **`effects/seed-pulse.ts` near `createPulseEffect`**: no dispose needed because state is a single primitive in the closure; if a future time effect holds rAF/listeners/timers, grow the interface with `dispose()` and have renderer's `destroy()` walk them.
- **`renderer.ts` near the subscribe callback**: slice-of-state-change is a recognized category (capture flare on owner flip, phase-change transitions). Inline diffing today; extract `watchSlice(getter, onChange)` only when N grows or predicates get complex.
- **`theme.ts` above `SEAT_COLORS`**: TODO about deriving variants from `base` via a color helper; mentions `colord` as candidate.
- (Phase 1 comments already in place: `renderer.ts`'s `applySnapshot`-is-sync contract and `destroy()` responsibilities checklist — no change.)

## Sequencing & commit plan

Exclusion-radius preview is **deferred** — seed-pulse is more architecturally interesting and more interesting game-wise. Two commits:

### Commit 1: tile layer + dirty-flag split + continuous-rAF mechanism

Renderer-only. Visual no-op.

- Allocate `tileLayer` (`OffscreenCanvas`) in `init`.
- Split `dirty` → `tilesDirty` + `frameDirty`.
- Split `requestRender` → `requestTilesRedraw` + `requestComposite`.
- Rename rAF callback `frame` → `renderFrame`.
- `renderFrame` body: rebuild `tileLayer` if `tilesDirty`, then blit + `drawHoverRing`.
- `if (animating) scheduleFrame()` seam in place; `animating = false` (no time effects yet) — hooks ready.
- `Ctx2D` type alias added to `draw.ts`; `drawAllTiles` and `drawTile` typed to it.
- Rename `applyLayout` → `computeLayout` in `layout.ts` (pure; no DOM mutation).
- Add closure-scoped `applyLayoutToSurfaces` in `init` (sets dims on main canvas + `tileLayer`); `relayout` calls it then `requestTilesRedraw`.
- `perfLog.timed('canvasDraw', ...)` now scopes the layer rebuild.

**Verification**: side-by-side commit jump. No visual diff. `wsRecv → paint` flat-or-better.

### Commit 2: seed-pulse

Store + theme + renderer + new module.

- Add `seeds: Array<{ coord, owner }>` to `DerivedState`; populate in `derived.ts` (refactor `computeExcludedCoords` to consume the list).
- Add `pulseFill` field to `SeatPalette`; hand-tune values; add `TODO` comment about `colord`.
- New `effects/seed-pulse.ts`: `createPulseEffect()` factory returning a `TimeEffect`-shaped object with `trigger / isActive / draw`.
- `renderer.ts`: instantiate `pulse`, add `prevTick`, wire trigger in subscribe (gated `phase === 'simulating'` && `tick !== prevTick`), wire `pulse.draw` and `pulse.isActive` in `renderFrame`.
- Encode the code comments listed above.

**Verification**: heartbeat visible during simulation phase only; truncates cleanly on next tick; no leaks across `key={matchId}` remount.
