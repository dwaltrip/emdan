# Match ↔ GameEngine split

Design note for refactoring `server/src/game/match.ts` into a thin orchestrator over a pure-ish `GameEngine`. Not implemented yet.

## Motivation

`match.ts` currently does four things in one class:

1. Socket I/O (send/receive, error messages, matchStarted/stateUpdate/matchEnded)
2. Timer / tick scheduling
3. Turn & phase state machine
4. Placement validation + simulation (expansion) logic

(1) and (2) are transport/lifecycle. (3) and (4) are game rules. They're entangled in one class, so:

- Rules can't be exercised without a WebSocket.
- Adding a new rule (new phase, new placement constraint, scoring variant) touches transport code.
- Error handling is a side effect (`sendError`) rather than a return value, which is harder to test and harder to localize later.

## Classification of current members

| Member | Category |
|---|---|
| `board`, `seedsRemaining`, `phase`, `currentTurn`, `tickNumber` | game state |
| `applyPlacement`, `expandSeeds`, `violatesExclusion`, `determineWinner`, `beginSimulation` | game rules |
| `id`, `startedAt`, `clientIds`, `timer`, `send`, `isConnected`, `onEnded`, `ended` | orchestration |
| `handleClientMessage`, `handleDisconnect`, `tick`, `finish`, `start`, `sendToAll`, `sendError`, `getSeatByClientId` | orchestration |
| `getSnapshot` | straddles — reads game state AND transport state (`connected`, `serverTimeMs`) |

One thing worth naming up front: **"`tick`" in the current code is misleading.** It's only a simulation step — placing phase has zero ticks. `GameEngine` owns `tickNumber` (recorded on every tile via `plantedTick`/`lastGrowthTick`); `Match` owns the *timer* that drives the step.

## The split

### `server/src/game/engine.ts` (new, ~180 lines)

Owns board, phase, turn, seeds, tickNumber. No sockets, no timers, no identity, no `Date.now()`.

```ts
export type PlaceErrorCode =
  | "wrong_phase"
  | "not_your_turn"
  | "out_of_bounds"
  | "no_seeds"
  | "tile_impassable"
  | "tile_occupied"
  | "too_close_to_seed";

export type PlaceResult =
  | { ok: true; phaseChanged?: MatchPhase }
  | { ok: false; code: PlaceErrorCode; message: string };

export type StepResult = { phaseChanged?: MatchPhase };

export interface GameStateSnapshot {
  tick: number;
  phase: MatchPhase;
  currentTurn: PlayerSeat | null;
  board: BoardState;  // tiles are deep-cloned — safe for Match to forward
  seats: Record<PlayerSeat, { occupiedTiles: number; seedsRemaining: number }>;
}

export interface GameEngineOptions {
  board?: Tile[][];  // optional pre-built board for deterministic tests
}

export class GameEngine {
  constructor(opts?: GameEngineOptions);

  get phase(): MatchPhase;

  placeSeed(seat: PlayerSeat, x: number, y: number): PlaceResult;
  step(): StepResult;   // no-op unless phase === "simulating"
  end(): void;          // externally triggered terminal state; idempotent

  toSnapshot(): GameStateSnapshot;
  determineWinner(): MatchWinner;
}
```

All rule checks live here: `wrong_phase`, `not_your_turn`, out-of-bounds, no-seeds, wall, occupied, radius. `violatesExclusion` becomes a module-local helper inside `engine.ts`. Coordinates are assumed to be integers — the protocol parser rejects non-integers upstream.

### `server/src/game/match.ts` (trimmed to ~130 lines)

Transport + orchestration only. No rule logic.

```ts
handleClientMessage(clientId, message) {
  if (this.ended) return;
  const seat = this.seatOf(clientId);
  if (!seat || message.type === "ping") return;

  const result = this.engine.placeSeed(seat, message.x, message.y);
  if (!result.ok) { this.sendError(seat, result.code, result.message); return; }

  this.broadcastState();
  if (result.phaseChanged === "simulating") this.startSimulationTimer();
}

private tick() {
  if (this.ended) return;
  const result = this.engine.step();
  this.broadcastState();
  if (result.phaseChanged === "ended") this.finish("boardFull", this.engine.determineWinner());
}

private toSnapshot(): MatchSnapshot {
  const gs = this.engine.toSnapshot();
  return {
    matchId: this.id,
    serverTimeMs: Date.now() - this.startedAt,
    tickIntervalMs: TICK_INTERVAL_MS,
    growthEveryTicks: GROWTH_EVERY_TICKS,
    tick: gs.tick, phase: gs.phase, currentTurn: gs.currentTurn, board: gs.board,
    players: {
      player1: { connected: this.isConnected("player1"), ...gs.seats.player1 },
      player2: { connected: this.isConnected("player2"), ...gs.seats.player2 },
    },
  };
}
```

`board.ts` and `blob.ts` stay untouched — they're primitives the engine composes.

## Ended state vs. match lifecycle

"Game is over" and "match is torn down" are related but distinct concepts. Three pieces of state, tracked in different places:

| Concept | Lives on | Means |
|---|---|---|
| `engine.phase === "ended"` | GameEngine | Rules-level: this game no longer accepts moves |
| `Match.ended === true` | Match | Lifecycle: this Match instance is torn down (timer cleared, cleanup done) |
| `MatchEndReason` (wire field) | Match → protocol | Why the match ended (`"disconnect"` \| `"boardFull"`) |

They correlate but don't collapse:

- **Board fills** → engine's own `step()` sets `phase = "ended"` and returns `phaseChanged: "ended"`. Match observes the return value and calls `finish("boardFull", ...)`, which sets `Match.ended = true`.
- **Disconnect** → Match sets `Match.ended = true` and calls `engine.end()` to keep engine state coherent. Match tags the wire message with `reason: "disconnect"`.
- **Future (forfeit / timeout / resign)** → same shape as disconnect. Match decides the match is over for transport reasons, tells the engine to lock via `engine.end()`, broadcasts with the appropriate reason.

The engine never needs to know *why* it ended. That's `MatchEndReason`, a wire-level field on `matchEnded`, owned by Match. The engine just needs to know *that* it ended, so its `toSnapshot()` stays truthful at broadcast time.

`engine.end()` is idempotent, so `finish()` can call it unconditionally — whether the engine ended itself via `step()` or is being ended externally, the final snapshot reflects `phase: "ended"`.

## Key design decisions

- **Engine owns `tickNumber`, not Match.** It's recorded on every tile via `plantedTick`/`lastGrowthTick`. Match owns the *timer* that calls `step()`.

- **Rule validation is the engine's job; transport validation stays upstream.** The engine checks `wrong_phase`, `not_your_turn`, bounds, wall, occupied, radius. Integer-ness of coordinates is pushed into `parseClientMessage` — it's a protocol concern, and once tightened there, `placeSeed(seat, x, y)` treats integers as a precondition.

- **Phase transitions come back as return values, not polling.** `placeSeed` and `step` return `{ phaseChanged?: MatchPhase }` on success. Match reacts causally (`if (result.phaseChanged === "simulating") startTimer()`), which reads better than polling `engine.phase` before/after and is robust to any future mutation that causes two transitions in one call.

- **`PlaceResult.code` is a union literal, not a free-form string.** `"wrong_phase" | "not_your_turn" | ...`. Zero runtime cost, gives exhaustive `switch` checking, and matches how codes are already treated on the wire. `message` stays free-form.

- **Engine never knows about disconnects.** Only that it's ended. `MatchEndReason` (disconnect/boardFull) is a wire-level field owned by Match. `engine.end()` is idempotent so Match can call it unconditionally from `finish()` regardless of whether `step()` already ended the engine.

- **`engine.toSnapshot()` deep-clones tiles.** Match forwards `gs.board` directly into `MatchSnapshot.board`; if the returned tiles aliased engine internals, any future Match-side transform could mutate the engine by accident. Cloning at the engine boundary is the cheap, local fix.

- **Engine wraps pure game state; Match wraps it into a `MatchSnapshot`.** Match adds `matchId`, `serverTimeMs`, the two timing constants, and per-seat `connected` status. Engine stays free of identity and wall-clock time — big win for tests.

- **Terrain can be injected via constructor options.** `new GameEngine({ board })` accepts a pre-built board (any size, any terrain layout); default is `createBoard()` with the protocol-level `GRID_WIDTH`/`GRID_HEIGHT`. Makes tests deterministic (no RNG in terrain generation) and lets tests use tiny boards (e.g. 5×5) to exercise edge cases cheaply.

- **Rule constants (`GROWTH_EVERY_TICKS`, `SEED_EXCLUSION_RADIUS`, `STARTING_SEEDS`) stay in `@shared/protocol`.** The client needs them (e.g. for future exclusion-halo rendering), so they're wire-level config, not engine-internal.

- **File name is `engine.ts`, class is `GameEngine`.** `engine` pairs naturally with `match`: Match orchestrates, GameEngine enforces rules. `GameEngine` over bare `Engine` because `this.engine = new GameEngine()` reads clearer at construction sites.

## Error code inventory

Current codes and where they move:

| Code | Today | After split |
|---|---|---|
| `invalid_coordinates` | Match (`Number.isInteger`) | `parseClientMessage` (drops the message entirely as malformed) |
| `out_of_bounds` | Match | GameEngine |
| `wrong_phase` | Match | GameEngine |
| `not_your_turn` | Match | GameEngine |
| `no_seeds` | Match | GameEngine |
| `tile_impassable` | Match | GameEngine |
| `tile_occupied` | Match | GameEngine |
| `too_close_to_seed` | Match | GameEngine |

Match becomes a pure forwarder for rule errors: `if (!result.ok) sendError(seat, result.code, result.message)`. Protocol-layer rejection (non-integer coords) stops silently at the parser — no wire error to surface, since a well-behaved client won't produce them.

## Testability payoff

The point of the split. With GameEngine isolated, tests look like:

```ts
const e = new GameEngine();
expect(e.placeSeed("player2", 10, 10)).toEqual({
  ok: false, code: "not_your_turn", message: expect.any(String),
});
expect(e.placeSeed("player1", 10, 10)).toEqual({ ok: true });
expect(e.phase).toBe("placing");
// ...exhaust seeds, expect phase === "simulating"
```

No WebSocket, no timer, no fake transport. Rule changes get unit tests for free.

## Minor cleanups to fold in during implementation

- **`determineWinner` currently reads from a `MatchSnapshot`** (line 387 of `match.ts`). After the move, it should read from the engine's own board / counts — no snapshot coupling.
- **Dead bounds re-check in `applyPlacement`** (line 159: `this.board[y]?.[x]`) is redundant once `isInsideBoard` has passed. Drop during the move.
- **Preserve ordering in `beginSimulation`**: current code broadcasts `stateUpdate` *before* setting the timer. New code must do the same — start-timer after broadcast.

## Open questions

- **Early win detection.** If a player's seeds go to 0 mid-simulation and the opponent clearly dominates, we still wait for board-fill. Out of scope here but worth a note for later.
- **Parameterized error messages.** `too_close_to_seed`'s message currently includes the radius value. Fine for now, but if we ever localize, we'll want `{ code: "too_close_to_seed"; radius: number }` and have Match format. Defer until there's a reason.
