## Non-obvious gotchas (not in the doc)

1. **Bootstrapping order.** An empty store has no players and no `activeInputPlayerId`. Until both are set (`addPlayer`, `setActiveInputPlayer`), no tile is `isPlaceableForActiveInputPlayer` — the board appears unresponsive. Easy to misdiagnose as "the store is broken."

2. **Tile components must use `useTileData`, not `useBoardState`.** `useBoardState` re-renders on every action — using it per tile silently defeats the whole per-tile architecture. The single biggest performance footgun.

3. **Store is per-instance, passed as a prop.** Created once via `createBlobWarsBoardStore(width, height)`, then handed down to components. Not a singleton, no module-level import.

4. **Cross-src import to verify.** `board-store.ts` imports `'../../../../shared/game-store/create-store'` — outside the React app's `src/`. Vite handles this at runtime, but TypeScript may complain if `tsconfig.json` has `rootDir`/`include` scoped to `src/` only. If type-check fails on that import: extend `include` to cover `../../shared`, set up a path alias, or move `shared/` into `ui/src/`.
